# 残り手順（コピペ用）

## 0. line_pending テーブル作成（新規マイグレーション）

Supabase SQL Editor で以下を実行:

```sql
CREATE TABLE IF NOT EXISTS terastar_line.line_pending (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID NOT NULL REFERENCES terastar_line.tenants(id) ON DELETE CASCADE,
  line_user_id TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(tenant_id, line_user_id)
);
CREATE INDEX IF NOT EXISTS idx_line_pending_tenant ON terastar_line.line_pending(tenant_id);
ALTER TABLE terastar_line.line_pending ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all for authenticated" ON terastar_line.line_pending FOR ALL USING (true);
```

## 1. Exposed schemas に terastar_line を追加

### 手順
1. [Supabase SQL Editor](https://supabase.com/dashboard/project/qxlucyxzfyqpmypmbokd/sql/new) を開く
2. 下記 SQL を**すべて選択してコピー**し、エディタに貼り付け
3. **Run** をクリック

### 実行する SQL（コピー用）

```
ALTER ROLE authenticator SET pgrst.db_schemas = 'public, logicpharm, payledger, terastar_line';
```

> **注意**: 既に `logicpharm`, `payledger` 以外のスキーマがある場合は、SQL 内のスキーマ一覧を編集してから実行してください。

## 2. LINE_CHANNEL_ACCESS_TOKEN を Vercel に設定

### Channel Access Token の場所（LINE Developers）
1. [LINE Developers - Messaging API設定](https://developers.line.biz/console/channel/2009302628/messaging-api) を開く
2. ページを**下にスクロール**
3. **「チャネルアクセストークン」** セクションを探す
4. **「チャネルアクセストークン（長期）」** の下に表示されている長い文字列がトークン
5. トークン右側の**コピーアイコン**（2つの四角が重なったマーク）をクリックしてコピー  
   ※ またはトークン文字列を選択して Cmd+C（Mac）／Ctrl+C（Win）でコピー

### Vercel への設定
- キー `LINE_CHANNEL_ACCESS_TOKEN` は**入力済み**です
- **値**の欄をクリックして、コピーしたトークンを **Cmd+V（Mac）／Ctrl+V（Win）** で貼り付け
- **Save** をクリック

## 4. 追加の環境変数（Vercel）

- `SUPABASE_SERVICE_ROLE_KEY`: Supabase → Project Settings → API → service_role（Webhook・Cron用）
- `LINE_CHANNEL_SECRET`: Webhook 署名検証用
- `CRON_SECRET`: （任意）フォローアップ Cron の認証用
