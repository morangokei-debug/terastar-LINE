# terastar-LINE セットアップ手順（未完了分）

## 1. LINE Webhook URL の保存（手動）

LINE Manager の保存ボタンは自動操作でクリックできないため、**手動で保存**してください。

1. [LINE Manager - Messaging API](https://manager.line.biz/account/@920bcvxk/setting/messaging-api) を開く
2. Webhook URL 欄に `https://terastar-line.vercel.app/api/line/webhook` が入力されていることを確認
3. **「保存」ボタンをクリック**

---

## 2. Supabase マイグレーションの実行

LogicPharm・PayLedger と競合しないよう、`terastar_line` スキーマを使用しています。

### 手順

1. [Supabase SQL Editor](https://supabase.com/dashboard/project/qxlucyxzfyqpmypmbokd/sql/new) を開く
2. **新規クエリ**を作成（+ ボタンまたは Cmd+N）
3. `supabase/migrations/20250303000000_initial_schema.sql` の内容を**すべてコピー**してエディタに貼り付け
4. **Run** をクリックして実行
5. エラーが出なければ完了

### 実行する SQL ファイル

```
supabase/migrations/20250303000000_initial_schema.sql
```

このファイルには以下が含まれます：

- `terastar_line` スキーマの作成
- テーブル作成（tenants, patients, prescriptions, follow_up_patterns, follow_up_schedules, follow_up_replies, chat_messages）
- RLS 有効化とポリシー
- 初期テナント「テラスターファーマシー」の登録

### 実行後の確認

- Supabase の **Table Editor** で `terastar_line` スキーマ内にテーブルが作成されていることを確認
- **API で利用する場合**：次の「3. Exposed schemas」を実行

---

## 3. Exposed schemas に terastar_line を追加（API から利用する場合）

Supabase の Data API で `terastar_line` スキーマを利用するには、Exposed schemas に追加する必要があります。

### 方法 A: SQL で実行（推奨）

1. [Supabase SQL Editor](https://supabase.com/dashboard/project/qxlucyxzfyqpmypmbokd/sql/new) を開く
2. `supabase/migrations/20250303000001_expose_terastar_line.sql` の内容をコピーして実行
3. **注意**: 既に `logicpharm`, `payledger` 以外のスキーマがある場合は、SQL 内のスキーマ一覧を編集してから実行してください

### 方法 B: ダッシュボードで追加

1. Supabase → **Project Settings**（歯車）→ **API**
2. **Exposed schemas** を探す
3. `terastar_line` を追加して保存

---

## 4. LINE_CHANNEL_ACCESS_TOKEN を Vercel に設定（メッセージ送信を使う場合）

1. [LINE Developers](https://developers.line.biz/console/) で該当チャネルの **Messaging API** を開く
2. **Channel access token** を発行またはコピー
3. [Vercel 環境変数](https://vercel.com/logicworks-projects/terastar-line/settings/environment-variables) で **Add** → キー `LINE_CHANNEL_ACCESS_TOKEN`、値にトークンを貼り付け → Save
